1. Recap: Week 1

1.7.1 Comparison Operators

Equal (==): True if the operands are equal

Strict Equal (===): True if the operands are equal and of the same type

1.7.2 Comparison Operators

Not equal (!=): Returns true if the operands are not equal

Strict not equal (!==): Returns true if the operands are of the same type but not equal, or are of different type

1.7.3 Comparison Operators

Greater than (>): True if the left operand is greater than the right operand.

Greater than or equal (>=): True if the left operand is greater than or equal to the right operand

1.7.4 Comparison Operators

Less than (<): True if the left operand is less than the right operand.

Less than or equal (<=): True if the left operand is less than or equal to the right operand

1.8 Logical Operators

Logical OR (||): Checks whether one of the two operands returns true once they have been evaluated

Logical AND (&&): Checks whether both operands returns true once they have been evaluated

6.0 Conventions & Terminology

6.1 Expressions & statements

A fragment of code that produces a value is called an expression. Every value that is written literally is called a statement.

6.2 Declaration vs assignment

Variable declaration creates a new variable wich can be assigned a value. A variable cannot be declared multiple times, but it can be assigned a value more than one if it was declared with const, let, or var keywords.

6.3 Naming conventions

  • Variable names and object keys should be camel-cased
  • Variables should use nouns as names, and functions should use verbs
  • Variables with array values should use plural nouns, variables with object values should use singular nouns
  • Variable and function names should preferably be short but expressive
